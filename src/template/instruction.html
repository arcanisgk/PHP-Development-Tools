HTTP Response Status Codes

HTTP response status codes indicate whether a specific HTTP request has completed successfully. Responses are grouped into five classes:

- Informational responses (100–199),
- Satisfactory responses (200–299),
- Redirects (300–399),
- Client errors (400–499),
- and server errors (500–599).

Status codes are defined in section 10 of RFC 2616. You can get the updated specifications in RFC 7231.

Informative Answers

100 Continue
This tentative response indicates that everything so far is fine and that the client should continue with the request or ignore it if it is already finished.

101 Switching Protocol
This code is sent in response to an Upgrade (en-US) request header by the client and indicates that the server accepts the user agent's proposed protocol change.

102 Processing(WebDAV(en - US))
This code indicates that the server has received the request and is still processing it, so no response is available.

103 Early Hints(en - US)
This status code is primarily intended to be used with the Link header, allowing the user agent to start preloading resources while the server prepares a response.

Satisfactory Answers

- GET: The resource has been obtained and is transmitted in the body of the message.
- HEAD: Entity headers are in the body of the message.
- PUT or POST: The resource that describes the result of the action is transmitted in the body of the message.
- TRACE: The body of the message contains the request message received by the server.

200 OK
The request has been successful.The meaning of a success varies depending on the HTTP method:

201 Created
The request was successful and a new resource was created as a result.This is typically the response sent after a PUT request.

202 Accepted
The request has been received, but has not yet been acted upon.It is a "no commitment" request, which means that there is no way in HTTP to send an asynchronous response indicating the result of processing the request.It is intended for cases where another process or server handles the request, or for batch processing.

203 Non - Authoritative Information
The request has been completed successfully, but its content has not been obtained from the originally requested source, but is instead collected from a local copy or from a third party.Except this condition, 200 OK response should be preferred instead of this response.

204 No Content(en - US)
The request has completed successfully but your response has no content, although headers can be useful.The user agent can update its cached headers for this resource with the new values.

205 Reset Content(en - US)
The request has been completed successfully, but its response has no content and in addition, the user agent has to initialize the page from which the request was made, this code is useful for example for pages with forms whose content must be deleted after the user sends it.

206 Partial Content
The request will partially serve the requested content.This feature is used by download tools like wget to continue the transfer of previously interrupted downloads, or to split a download and process the parts simultaneously.

207 Multi - Status(WebDAV(en - US))
A Multi - State response conveys information about multiple resources in situations where multiple status codes might be appropriate.The body of the request is an XML message.

208 Multi - Status(WebDAV(en - US))
The list of DAV elements has already been previously notified, so they will not be listed again.

226 IM Used(HTTP Delta encoding)
The server has fulfilled a GET request for the resource, and the response is a representation of the result of one or more instance manipulations applied to the current instance.

Redirects

300 Multiple Choice(en - US)
This request has more than one possible response.User - Agent or the user must choose one of them.There is no standardized way to select one of the answers.

301 Moved Permanently(en - US)
This response code means that the URI of the requested resource has been changed.Probably a new URI will be returned in the response.

302 Found
This response code means that the resource of the requested URI has been temporarily changed.new URI changes will be added in the future.Therefore, the same URI must be used by the client in future requests.

303 See Other(en - US)
The server sends this response to direct the client to a new requested resource at another address using a GET request.

304 Not Modified
This is used for "caching" purposes.Indicates to the client that the response has not been modified.Then the client can continue to use the same version stored in its cache.

305 use Proxy
It was defined in a previous version of the HTTP protocol specification to indicate that a requested response should be accessed from a proxy.It has been deprecated due to security concerns related to setting up a proxy.

306 unused
This response code is no longer used.It is currently reserved.It was used in previous versions of the HTTP1.1 specification.

307 Temporary Redirect(en - US)
The server sends this response to direct the client to get the requested resource at another URI with the same method used in the previous request.It has the same semantics as the HTTP 302 Found response code, with the exception that the user agent must not change the HTTP method used: if a POST was used in the first request, another POST must be used in the second request.

308 Permanent Redirect(en - US)
It means that the resource is now permanently located at another URI, specified by the Location: HTTP header response.It has the same semantics as the HTTP 301 Moved Permanently response code, with the exception that the user agent must not change the HTTP method used: if a POST was used in the first request, another POST must be used in the second request.

Client Errors

400 Bad Request
This response means that the server was unable to interpret the request due to invalid syntax.

401 Unauthorized
Authentication is required to obtain the requested response.This is similar to 403, but in this case, authentication is possible.

402 Payment Required
This response code is reserved for future use. The initial objective of creating this code was to be used in digital payment systems.However, it is not currently being used.

403 Forbidden
The client does not have the necessary permissions for certain content, so the server is refusing to provide an appropriate response.

404 Not Found
The server could not find the requested content.This response code is one of the most famous given its high occurrence on the web.

405 Method Not Allowed(en - US)
The requested method is known to the server but has been disabled and cannot be used.The two mandatory methods, GET and HEAD, should never be disabled and should not return this error code.

406 Not Acceptable(en - US)
This response is sent when the server, after applying a server - driven content negotiation, does not find any content following the criteria given by the user.

407 Proxy Authentication Required(en - US)
This is similar to the 401 code, but the authentication must be done through a proxy.

408 Request Timeout
This response is sent on an idle connection on some servers, even without any prior request from the client.It means that the server wants to disconnect this unused connection.This response is widely used since some browsers, like Chrome, Firefox 27 +, or IE9, use HTTP pre - connection mechanisms to speed up browsing.Also note that some servers simply disconnect the connection without sending this message.

409 Conflict(en - US)
This response can be sent when a request conflicts with the current state of the server.

410 Gone(en - US)
This response can be sent when the requested content has been deleted from the server.

411 Length Required(en - US)
The server rejects the request because the Content - Length header field is not defined and the server requires it.

412 Precondition Failed(en - US)
The client has indicated pre - conditions in its headers which the server does not meet.

413 Payload Too Large
The request entity is longer than the limits defined by the server; the server may close the connection or return a Retry - After header field.

414 URI Too Long(en - US)
The URI requested by the client is longer than the server is willing to interpret.

415 Unsupported Media Type(en - US)
The media format of the requested data is not supported by the server, so the server rejects the request.

416 Requested Range Not Satisfiable(en - US)
The range specified by the Range header field in the request does not match; it is possible that the range is outside the target data size of the URI.

417 Expectation Failed(en - US)
It means that the expectation indicated by the Requested Expect header field cannot be fulfilled by the server.

418 I'm a teapot
The server refuses to try to make coffee with a kettle.

421 Misdirected Request
The request was directed to a server that is not capable of producing a response. This may be sent by a server that is not configured to produce responses for the combination of the scheme and authority that are included in the requested URI

422 Unprocessable Entity (en-US) (WebDAV (en-US))
The request was well formed but could not be followed due to semantic errors.

423 Locked (WebDAV (en-US))
The resource being accessed is locked.

424 Failed Dependency (WebDAV (en-US))
The request failed due to a failure of a previous request.

426 Upgrade Required (en-US)
The server refuses to fulfill the request using the current protocol but may be willing to do so after the client upgrades to a different protocol. The server sends an Upgrade header in a response to indicate the required protocols.

428 Precondition Required (en-US)
The origin server requires that the request be conditional. It is intended to prevent 'missing update' problems, where a client GETS a resource state, modifies it, and PUT it back to the server, while a third party has modified the server state, leading to a conflict.

429 Too Many Requests (en-US)
The user has sent too many requests in a given period of time.

431 Request Header Fields Too Large (en-US)
The server is unwilling to process the request because the header fields are too long. The request MAY be re-uploaded after reducing the size of the requested header fields.

451 Unavailable For Legal Reasons (en-US)
The user requests an illegal resource, such as a government-censored web page.

Server Errors

500 Internal Server Error
The server has encountered a situation that it does not know how to handle.

501 Not Implemented (en-US)
The requested method is not supported by the server and cannot be handled. The only methods that servers require support (and therefore should not return this code) are GET and HEAD.

502 Bad Gateway
This error response means that the server, while working as a gateway to get a response needed to handle the request, got an invalid response.

503 Service Unavailable
The server is not ready to handle the request. Common causes could be that the server is down for maintenance or is overloaded. Note that along with this response, a user-friendly page explaining the problem must be submitted. These responses should be used for temporary conditions and the HTTP Retry-After: header should, if possible, contain the estimated time before service retrieval. The webmaster should also take care of the cache-related headers that are sent along with this response, as these temporary condition responses should usually not be in the cache.

504 Gateway Timeout
This error response is given when the server is acting as a gateway and cannot get a response in time.

505 HTTP Version
Not Supported The version of HTTP used in the request is not supported by the server.

506 Variant
Also Negotiates (en-US) The server has an internal configuration error: transparent content negotiation for the request results in a circular reference.

507 Insufficient Storage (en-US)
The server has an internal configuration error: the chosen resource variable is configured to engage the transparent content negotiation itself, and is therefore not a suitable endpoint for the negotiation process.

508 Loop Detected (en-US) (WebDAV (en-US))
The server detected an infinite loop while processing the request.

510 Not Extended (en-US)
Additional extensions to the request are required for the server to fulfill them.

511 Network Authentication Required (en-US)
Status code 511 indicates that the client needs to authenticate to gain access to the network.
